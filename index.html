HOW JAVASCRIPT WORKS BEHIND THE SCENES

Every time code is being given to the browser, the browser will interpret the code with a built-in JavaScript engine. That engine will use a parser, reading the code line by line, checking for validity. If there are any errors found, it will stop the execution and return an error.

If there are no issues, the parser will create an abstract syntax tree, which will be converted to machine code and then be actually run.

Execution Context

The execution context is where a piece of code is executed. The default execution context is the Global Execution Context, which is code that is not inside of any function and associated with the global object.

Code that are in functions will get their own Execution Contexts when they are called. As they are called, they will be added to an Execution Stack. Until a function in that stack is completed, it will remain in the stack. When the function is completed, it will be taken off the stack.

There are two phases for the Execution Context:
The creation phase, and the Execution Phase

In the creation phase, the argument object is created, containing all the arguments that were passed into the function.

Code is scanned for function declarations: for each function, a property is created in the Variable Object, pointing to the function.

Code is scanned for variable declarations: for each variable, a property is created in the Variable Object, and set to undefined.

This process is referred to as hoisting.

The scope chain does something similarly! If there is a variable that is found in a function that is calling a variable not in that function, it will look outwards to the parent, and then from there the global scope.
